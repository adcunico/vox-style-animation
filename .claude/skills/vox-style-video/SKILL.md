# Vox Style Video

Turn a topic into a 30-second Vox-style editorial paper-collage explainer, generated as three sequential 10-second scenes with locked keyframes.

## Prerequisites

- `KIE_API_KEY` in a `.env` file at the repo root (get one at https://kie.ai/api-key). Budget roughly 400-700 credits per finished video including retries.
- `ffmpeg` on PATH.
- Node 18+.

## Core method: keyframe pairs, not free generation

Never let the video model invent compositions. For every scene:

1. Generate (and get the user's approval on) a START image and an END image first — images are cheap, videos are expensive.
2. Give Gemini Omni both images: first = exact frame 0, second = exact final frame, with 4 timed beats describing the transformation.
3. The END frame of scene N is generated by image-to-image FROM scene N-1's last frame, so the paper world stays pixel-consistent.
4. Scene N's video starts from scene N-1's actual extracted last frame.

Do not pass three keyframes to one clip — with 3+ images Omni treats them as loose references and redraws faces and text. If a scene needs a mid keyframe (e.g. a camera pan), split it into two clips (4s + 6s) that share the mid frame, and join them with a 0.25s crossfade.

## Hard rules (learned the expensive way)

- NEVER put a famous person's name in a VIDEO prompt — Gemini's safety filter blocks it ("prominent public figure"). Names and titles may only appear as text baked into keyframe images (GPT Image 2 renders text reliably and allows names).
- Photographic likenesses of famous people in input images also block. Depict real people as hand-painted / folk-art style portraits — those pass.
- People are static artwork: faces never move, blink, or breathe. Cutouts move only as rigid paper pieces (slide, stamp, tilt, fade, rack focus).
- All videos are silent (`--no-audio`); narration is recorded separately (e.g. ElevenLabs) and laid over the stitched master in post.
- Composition budget: max 3 major elements at full contrast at once; a third of the frame stays empty paper; when a new element enters, an old one exits.
- Every clip: hold frame 0 still for 0.5s; end settled and still (final scene holds a full second).

## Workflow

1. **Script**: write the narration as 3 chunks of <=10s each (~35 spoken words per chunk): hook -> twist -> payoff. Save sections in `storyboard/<project>-sections.json`, config in `storyboard/<project>.config.json` (copy the frida-kahlo ones as templates; update the default config path in the script or pass `--config`).
2. **Style**: adapt `VOX-STYLE.md` — palette, motifs, and one visual through-line that pays off in scene 3 (e.g. the gold crack that blooms with flowers).
3. **Prompts**: write per-scene image and video prompts in `prompts/<project>/` following the frida-kahlo examples (they encode all rules above).
4. **Generate** (from the repo root):
   ```bash
   node scripts/kie-sequential-video.mjs generate-first-image
   node scripts/kie-sequential-video.mjs generate-end-image --scene 01 --legacy-upload
   node scripts/kie-sequential-video.mjs generate-scenes --scene 01 --no-audio --legacy-upload --end-frame
   # repeat end-image + scene for 02 and 03, each end frame conditioned on the previous last frame via --start-frame
   ```
   Show the user every keyframe image before rendering its video.
5. **Stitch**: join scenes with 0.35s video crossfades (offsets 9.65 and 19.30 for three 10s clips), `-an`, H.264 CRF 18. See `VOX-SEQUENTIAL-SCENE-WORKFLOW.md` for the full ffmpeg command.
6. **Post**: record narration (three chunks map 1:1 onto the three sections), lay it over the master with ffmpeg, add any exact text overlays that were left as blank cards.
